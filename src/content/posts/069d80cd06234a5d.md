Granblue Fantasy: Relink（碧蓝幻想：Relink）中敌人的 OD（Overdrive）状态控制脚本。两个版本：带 Lua 定时器的自动切换版（CT），以及纯手动控制版（TXT）。

![封面](/img/projects/069d80cd06234a5d/cover.jpg)

## 什么是 OD

OD（Overdrive）是敌人在战斗中进入的强化状态。OD 状态下敌人攻击模式改变，部分敌人会释放强力技能。OD 槽满时（state=3）触发 Break 机会——打空 OD 槽可使敌人进入 Break 昏厥状态。

此脚本通过注入敌人状态更新函数，直接操控 OD 状态值和 OD 点数，实现自动/手动 OD 控制。

## 注入点

两个 Hook 点位于 `granblue_fantasy_relink.exe`：

| Hook | 地址 | 功能 |
|------|------|------|
| od_hook | `exe+1F7123F` | 敌人状态更新入口，劫持 `mov rcx, [r12+0x338]` |
| od_hook2 | `exe+1F71271` | OD 槽值读取，劫持 `mov rcx, [r12+0x270]` |

### Hook1 — 状态写入

```
原始代码:
  mov rcx, [r12+0x338]   ; r12 = enemy_state_ptr
  mov rax, [rcx]          ; 虚表取函数指针

劫持后跳转到 od_hook，根据 ctrlFlag 修改:
  [rcx+0xDC8] = OD 状态 (0=正常, 1=?, 3=满OD)
  [rcx+0xDD0] = OD 点数 (默认 300000 / 0x493E0)
```

### Hook2 — OD 槽监控

```
原始代码:
  mov rcx, [r12+0x270]

劫持后先记录 xmm7（OD 槽当前值）到 [od_gauge_value]
```

## ctrlFlag 控制模式

| ctrlFlag | 状态 | 效果 |
|----------|------|------|
| `0` | — | 关闭，不干预 |
| `1` | OD | 强制状态=1，点数=od_points |
| `2` | OD 锁点 | 仅当已是 OD（`[rcx+0xDC8]==1`）时锁点数，不改变状态 |
| `3` | 普通 | 强制状态=0（永不开 OD），点数=od_points |
| `4` | OD 满 | 状态=3（gauge 计算直接返回 1.0），点数清零 |

### 汇编逻辑

```
ctrlFlag=0 → od_pass（跳过）
ctrlFlag=2 → 检查 [rcx+0xDC8]==1，不满足则跳过
ctrlFlag=4 → [rcx+0xDC8]=3, [rcx+0xDD0]=0（满 OD + 点数清零）
其他值(1/3) → [rcx+0xDC8]=ctrlFlag, [rcx+0xDD0]=od_points
```

## 版本一：自动切换 CT 脚本

`疯狗ODv1.0.CT` — 带 Lua 定时器，1 秒 ON / 9 秒 OFF 循环：

```asm
[ENABLE]
alloc(od_hook, 256)
alloc(od_enemy_state_ptr, 8)
alloc(od_gauge_value, 4)
alloc(ctrlFlag, 4)
alloc(od_points, 4)
label(od_hook_ret)
label(od_hook2)
label(od_hook2_ret)
label(od_null)
label(od_pass)
registersymbol(od_enemy_state_ptr)
registersymbol(od_gauge_value)
registersymbol(ctrlFlag)
registersymbol(od_points)

granblue_fantasy_relink.exe+1F7123F:
  jmp od_hook
  nop
  nop
  nop
od_hook_ret:

granblue_fantasy_relink.exe+1F71271:
  jmp od_hook2
  nop
  nop
  nop
od_hook2_ret:

od_hook:
  mov [od_enemy_state_ptr], r12
  mov rcx, [r12+0x338]
  test rcx, rcx
  jz od_null
  cmp dword ptr [ctrlFlag], 0
  je od_pass
  // 写状态
  mov eax, [ctrlFlag]
  cmp eax, 2
  jne od_store_status
  // ctrlFlag=2: 只锁点数不锁状态
  cmp dword ptr [rcx+0xDC8], 1
  jne od_pass
od_store_status:
  cmp eax, 4
  jne od_not_mode4
  // ctrlFlag=4: 强锁满(状态=3)
  mov dword ptr [rcx+0xDC8], 3
  mov dword ptr [rcx+0xDD0], 0
  jmp od_pass
od_not_mode4:
  mov [rcx+0xDC8], eax
  mov eax, [od_points]
  mov [rcx+0xDD0], eax
od_pass:
  mov rax, [rcx]
  jmp od_hook_ret
od_null:
  xor eax, eax
  jmp od_hook_ret

od_hook2:
  movss [od_gauge_value], xmm7
  mov rcx, [r12+0x270]
  jmp od_hook2_ret

ctrlFlag:
  dd 0
od_points:
  dd 493E0

{$lua}
-- 清理旧定时器
if g_myTimer ~= nil then
  g_myTimer.destroy()
  g_myTimer = nil
end

-- 初始状态：接下来要写入 4
local nextValue = 4

-- 创建定时器
g_myTimer = createTimer(nil)
g_myTimer.Interval = 1000 -- 1秒后触发第一次切换
g_myTimer.OnTimer = function(timer)
  if not getAddress("ctrlFlag") then return end

  -- 写入当前设定的值
  writeInteger("ctrlFlag", nextValue)
  print(string.format("[定时器] ctrlFlag 已切换为: %d", nextValue))

  -- 动态调整下一次切换的时间
  if nextValue == 4 then
    timer.Interval = 1000  -- 4 状态维持 1 秒
    nextValue = 0          -- 下一次切换的目标变为 0
  else
    timer.Interval = 9000  -- 0 状态维持 9 秒
    nextValue = 4          -- 下一次切换的目标变为 4
  end
end
{$asm}

[DISABLE]
granblue_fantasy_relink.exe+1F7123F:
  mov rcx, [r12+0x338]
  mov rax, [rcx]

granblue_fantasy_relink.exe+1F71271:
  mov rcx, [r12+0x270]

{$lua}
-- 关闭脚本时销毁定时器
if g_myTimer ~= nil then
  g_myTimer.destroy()
  g_myTimer = nil
  print("[定时器] 已成功关闭")
end
{$asm}

unregistersymbol(od_gauge_value)
unregistersymbol(od_enemy_state_ptr)
unregistersymbol(ctrlFlag)
unregistersymbol(od_points)
dealloc(od_hook)
dealloc(od_enemy_state_ptr)
dealloc(od_gauge_value)
dealloc(ctrlFlag)
dealloc(od_points)
```

### 自动切换逻辑

```
时间线:
  [0s] ctrlFlag=4 (强锁满, 持续1s)
  [1s] ctrlFlag=0 (关闭, 持续9s)
  [10s] ctrlFlag=4 (循环)
  ...

敌人每 10 秒进入一次满 OD 状态持续 1 秒，
其余 9 秒正常运作。
```

在 CE 中可通过修改 `od_points` 变量调整 OD 点数（默认 `0x493E0` = 300,000），或手动设置 `ctrlFlag` 切换模式。

## 注册变量

脚本注册 4 个可读写变量到 CE 地址列表：

| 变量 | 类型 | 含义 |
|------|------|------|
| `od_enemy_state_ptr` | qword | 当前敌人状态对象指针（r12） |
| `od_gauge_value` | float | 当前 OD 槽值（xmm7，来自 Hook2） |
| `ctrlFlag` | dword | 控制模式：0=关, 1=OD, 2=锁点, 3=普通, 4=满OD |
| `od_points` | dword | OD 点数写入值（默认 0x493E0，约 30 万） |

## 偏移说明

| 偏移 | 类型 | 含义 |
|------|------|------|
| `[r12+0x338]` | ptr | 敌人状态对象（null 检查） |
| `[rcx+0xDC8]` | dword | OD 状态（0=正常, 3=满 OD） |
| `[rcx+0xDD0]` | dword | OD 点数（累积值） |
| `[r12+0x270]` | ptr | OD 槽 UI 读取源 |

## 使用方法

### 1. 导入脚本

- **CT 文件**：双击 `疯狗ODv1.0.CT` 或 CE 菜单 `File → Load` 加载
- **TXT 文件**：打开 Cheat Engine → `Ctrl+Alt+A` 打开 Auto Assemble → 粘贴 `od.txt` 全部内容 → `File → Assign to current cheat table`

### 2. 附加进程

确保 CE 已附加到 `granblue_fantasy_relink.exe`（`File → Open Process`）。

### 3. 启用脚本

- 勾选 CT 表中的脚本条目
- **CT 版**：自动开始 1s/9s 循环，Lua 控制台可见切换日志
- **TXT 版**：默认 `ctrlFlag=0`（关闭状态），需手动修改 `ctrlFlag` 值

### 4. 手动控制（CT 版也可用）

在 CE 地址列表中修改 `ctrlFlag`：

```
ctrlFlag = 0  → 关闭（不干预）
ctrlFlag = 1  → 强制进入 OD（状态=1，点数=od_points）
ctrlFlag = 2  → 已是 OD 时锁点数，不改变状态
ctrlFlag = 3  → 强制普通状态（永不开 OD）
ctrlFlag = 4  → 强制满 OD（状态=3，点数清零）
```

修改 `od_points` 可调整点数上限。

### 5. 关闭

**脚本启用后不可关闭。** ctrlFlag=0 可伪关闭，最好是直接重启游戏。

> **注意**：游戏版本更新可能导致注入地址偏移变化。当前地址适配 v1.3.2。若失效需重新定位 `r12+0x338` 和 `r12+0x270` 的引用点。
