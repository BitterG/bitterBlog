# 读写锁-锁降级

<font style="color:#74B602;">一锁两用</font>，一个资源可以被<font style="color:#74B602;">多个读线程</font>访问，或一<font style="color:#74B602;">个写线程</font>访问。读多写少场景最为适用，性能较高

<font style="color:#DF2A3F;">缺点</font>：<font style="color:#DF2A3F;">1.写锁饥饿问题</font>（读特别多，写特别少时，会导致写大概率抢不到锁）<font style="color:#DF2A3F;">	2.锁降级</font>

## 可重入读写锁ReentrantReadWriteLock
```java
class MyDate {
    Map<Integer, String> map = new HashMap<>();	//用来存储数据

    Lock lock = new ReentrantLock();	//锁

    ReentrantReadWriteLock rwLock = new ReentrantReadWriteLock();	//读写锁

    public void write(Integer key, String val) {
//        lock.lock();
        rwLock.writeLock().lock();	//获取写锁
        try {
            System.out.println(Thread.currentThread().getName() + "正在写入");
            try {TimeUnit.MILLISECONDS.sleep(500L);} catch (InterruptedException e) {e.printStackTrace();}
            map.put(key, val);
            System.out.println(Thread.currentThread().getName() + "写入完毕");
        } catch (Exception e) {
            e.printStackTrace();
        } finally {
//            lock.unlock();
            rwLock.writeLock().unlock();	//释放写锁
        }
    }

    public String read(Integer key) {
//        lock.lock();
        rwLock.readLock().lock();	//获取读锁
        String temp = "";
        try {
            System.out.println(Thread.currentThread().getName() + "正在读取");
            try {TimeUnit.MILLISECONDS.sleep(200L);} catch (InterruptedException e) {e.printStackTrace();}
            temp = map.get(key);
            System.out.println(Thread.currentThread().getName() + "读取完毕" + temp);
        } catch (Exception e) {
            e.printStackTrace();
        } finally {
//            lock.unlock();
            rwLock.readLock().unlock();	//释放读锁
        }
        return temp;
    }
}

public class ReentranReadWriteLock {
    public static void main(String[] args) {
        MyDate myDate = new MyDate();

        //10个线程写入数据
        for (int i=0; i < 10; i++) {
            int finalI = i;
            new Thread(() -> {
                myDate.write(finalI, "数据" + finalI);
            }).start();
        }
    	//10个线程读取数据
        for (int i=0; i < 10; i++) {
            int finalI = i;
            new Thread(() -> {
                myDate.read(finalI);
            }).start();
        }
    }
}
```

先完成写，之后写锁共享数据一起读取。<font style="color:#117CEE;">读锁未释放写锁无法获取锁</font>，反之亦然。

## 锁降级
ReentrantReadWriteLock<font style="color:#ED740C;">锁降级</font>: 将<font style="color:#ED740C;">写入锁降级为读锁</font>(类似Linux文件读写权限理解，就像写权限要高于读权限一样锁的严苛程度变强叫做升级，反之叫做降级。<font style="color:#DF2A3F;">写锁可以降级，但不能升级。（悲观锁策略）</font>



<font style="color:#1DC0C9;">写锁的降级，降级成为了读锁</font>

<font style="color:#1DC0C9;">1 .</font>如果同一个线程持有了写锁，在没有释放写锁的情况下，它还可以继续获得读锁。这就是写锁的降级，降级成为了读锁.

<font style="color:#1DC0C9;">2 .</font>规则惯例，先获取写锁，然后获取读锁，再释放写锁的 次序。

<font style="color:#1DC0C9;">3 .</font>如果释放了写锁，那么就完全转换为读锁。

<font style="color:#117CEE;">锁降级是为了让线程感知数据变化，目的是保证数据可见性</font>

![图片](/img/juc/e4030a2e7ebb14aa/1674910346043-9c987027-7288-4f9c-8ece-6eeca3d793d8-423710.webp)

```java
public class LockDown {
    ReentrantReadWriteLock rwLock = new ReentrantReadWriteLock();

    //一个线程
    public static void main(String[] args) {
        LockDown lockDown = new LockDown();

        ReentrantReadWriteLock.ReadLock readLock = lockDown.rwLock.readLock();
        ReentrantReadWriteLock.WriteLock writeLock = lockDown.rwLock.writeLock();

        writeLock.lock();   //获取写锁
        System.out.println("写");
        readLock.lock();    //获取读锁-降级为读锁
        System.out.println("读");
        writeLock.unlock(); //释放写锁
        readLock.unlock();  //释放读锁
    }
}
```

## StampedLock-标记锁
StampedLock采用乐观锁形式，其他线程尝试获取锁时不会被阻塞，是对读锁的优化，因此，在获取乐观读锁后，需要对结果进行校验（读锁占用时，写锁可介入）（不可重入）

```java
public class StampedLockDemo {
    static StampedLock stampedLock = new StampedLock();

    public void write() {
        long stamp = stampedLock.writeLock();	//获取锁和标记
        try {
            System.out.println("写操作");
        } finally {
            stampedLock.unlockWrite(stamp);	//释放锁
        }
    }
    public void read() {
        long stamp = stampedLock.readLock();
        try {
            System.out.println("读操作");
        } finally {
            stampedLock.unlockRead(stamp);
        }
    }
}
```

```java
public class StampedLockDemo {
    static StampedLock stampedLock = new StampedLock();

    //使用乐观读锁
    public void tryOptimisticRead() {
        long stamp = stampedLock.tryOptimisticRead();   //乐观读锁
        try {
            System.out.println("读操作");
        } catch (Exception e) {e.printStackTrace();} finally {}
        //validate(stamp)判断 返回值true没被修改 false被修改
        if (!stampedLock.validate(stamp)) {
            //读取失败 变为悲观读 传统方式
            stamp = stampedLock.readLock();
            try {
                System.out.println("读操作");
            } finally {
                stampedLock.unlockRead(stamp);
            }
        }
        System.out.println("期间没有被修改乐观读取成功");
    }

    public static void main(String[] args) {}
}
```

## StampedLock缺点
1.不支持重入

2.StampedLock的悲观锁和写锁都不支持条件变量（Condition）

3.使用Stamped一定不要调用中断操作，即不要调用interrupt()方法
