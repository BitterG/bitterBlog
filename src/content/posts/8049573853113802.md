# 缓冲区get/put方法

### <font style="color:rgb(18, 18, 18);">get() 方法</font>
**<font style="color:rgb(18, 18, 18);">ByteBuffer 类中有四个 get() 方法：</font>**

<font style="color:rgb(99, 99, 99);">1）byte get();  
</font><font style="color:rgb(99, 99, 99);">2）ByteBuffer get( byte dst[] );  
</font><font style="color:rgb(99, 99, 99);">3）ByteBuffer get( byte dst[], int offset, int length );  
</font><font style="color:rgb(99, 99, 99);">4）byte get( int index );</font>

<font style="color:rgb(18, 18, 18);">第一个方法获取单个字节。第二和第三个方法将一组字节读到一个数组中。第四个方法从缓冲区中的特定位置获取字节。那些返回 ByteBuffer 的方法只是返回调用它们的缓冲区的 this 值。</font>

<font style="color:rgb(18, 18, 18);">此外，我们认为前三个 get() 方法是相对的，而最后一个方法是绝对的。 相对 意味着 get() 操作服从 limit 和 position 值 ― 更明确地说，字节是从当前 position 读取的，而 position 在 get 之后会增加。另一方面，一个 绝对 方法会忽略 limit 和 position 值，也不会影响它们。事实上，它完全绕过了缓冲区的统计方法。</font>

<font style="color:rgb(18, 18, 18);">上面列出的方法对应于 ByteBuffer 类。其他类有等价的 get() 方法，这些方法除了不是处理字节外，其它方面是是完全一样的，它们处理的是与该缓冲区类相适应的类型。</font>

<font style="color:rgb(18, 18, 18);"></font>

**<font style="color:rgb(18, 18, 18);">ByteBuffer 类中有五个 put() 方法：</font>**

> <font style="color:rgb(99, 99, 99);">1）ByteBuffer put( byte b );  
</font><font style="color:rgb(99, 99, 99);">2）ByteBuffer put( byte src[] );  
</font><font style="color:rgb(99, 99, 99);">3）ByteBuffer put( byte src[], int offset, int length );  
</font><font style="color:rgb(99, 99, 99);">4）ByteBuffer put( ByteBuffer src );  
</font><font style="color:rgb(99, 99, 99);">5）ByteBuffer put( int index, byte b );</font>
>

<font style="color:rgb(18, 18, 18);">与 get() 方法一样，我们将把 put() 方法划分为 相对 或者 绝对 的。前四个方法是相对的，而第五个方法是绝对的。</font>
