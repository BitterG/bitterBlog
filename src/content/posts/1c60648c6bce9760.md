基于 Go 实现的轻量级微服务网关，支持路由转发、认证鉴权、限流、负载均衡、熔断、可观测性等核心功能，使用 Docker Compose 一键部署。

> 项目地址：[https://gitee.com/gourd_bitter/bitter-gate](https://gitee.com/gourd_bitter/bitter-gate)

## 架构

```
                        ┌─────────────────────────────────┐
                        │           BitterGate            │
                        │                                 │
Client ─── HTTP ───────▶│  TraceID → Logging → Router    │
                        │              │                  │
                        │        ┌─────▼──────┐           │
                        │        │  JWT 认证   │           │
                        │        │  RBAC 鉴权  │           │
                        │        │  令牌桶限流  │           │
                        │        │  熔断器     │           │
                        │        │  头部改写   │           │
                        │        └─────┬──────┘           │
                        │             │ 反向代理           │
                        └─────────────┼───────────────────┘
                                      │
               ┌──────────────────────┼──────────────────┐
               ▼                      ▼                   ▼
          user-a:9001           user-b:9001          order:9002
          (上游服务)             (上游服务)            (上游服务)

Prometheus(:9091) ──抓取指标──▶ Prometheus(:9090) ──▶ Grafana(:3000)
```

## 功能列表

| 功能 | 说明 |
|------|------|
| 前缀路由匹配 | 基于前缀树，支持多路由配置，最长前缀优先 |
| 反向代理 | 透明转发，自动附加 X-Forwarded-* 头 |
| JWT 认证 | 登录签发 Token，请求携带 Bearer Token 验证 |
| RBAC 权限控制 | 基于 JWT roles 的路由级别访问控制 |
| 令牌桶限流 | 每个路由独立配置限流速率，按客户端 IP 隔离 |
| 轮询负载均衡 | 多上游实例间轮询分发请求 |
| 上游健康检查 | 定时探活，自动摘除不健康实例并恢复 |
| 熔断器 | 三态熔断（Closed/Open/Half-Open），失败3次熔断，冷却后自动探测恢复 |
| 配置热更新 | 监听 Routers.json 文件变化，无需重启自动生效 |
| 请求/响应头改写 | 每条路由可配置注入自定义请求头和响应头 |
| Trace ID | 每请求生成唯一 ID，透传上游，响应头返回客户端 |
| 结构化日志 | slog JSON 格式，含 trace_id、路径、状态码、耗时 |
| Prometheus 指标 | 请求计数、延迟直方图（P99）、状态码分布，独立端口暴露 |
| Grafana 面板 | 预置可视化面板，Docker Compose 一键启动 |
| bcrypt 密码存储 | 用户密码哈希存储 |
| Docker 部署 | 多阶段构建镜像，Docker Compose 一键启动 |

## 快速启动

### 前置条件
- Docker & Docker Compose

### 启动

```bash
git clone <repo-url>
cd BitterGate

docker compose up --build -d
```

服务地址：
- 网关：`http://localhost:8080`
- Grafana：`http://localhost:3000`
- Prometheus：`http://localhost:9090`

### 停止

```bash
docker compose down
```

## 接口示例

### 登录获取 Token

```bash
STUDENT_TOKEN=$(curl -s -X POST http://localhost:8080/login \
  -H "Content-Type: application/json" \
  -d '{"username":"kugua","password":"123456"}' \
  | grep -o '"token":"[^"]*"' | cut -d'"' -f4)

ADMIN_TOKEN=$(curl -s -X POST http://localhost:8080/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"123456"}' \
  | grep -o '"token":"[^"]*"' | cut -d'"' -f4)
```

### 访问受保护路由

```bash
# student 可访问 /user/*
curl -H "Authorization: Bearer $STUDENT_TOKEN" http://localhost:8080/user/profile

# student 无权访问 /order/*（返回 403）
curl -H "Authorization: Bearer $STUDENT_TOKEN" http://localhost:8080/order/detail

# admin 可访问 /order/*
curl -H "Authorization: Bearer $ADMIN_TOKEN" http://localhost:8080/order/detail
```

### 健康检查

```bash
curl http://localhost:8080/healthz
# {"status":"ok"}
```

### 查看当前路由表（仅 admin）

```bash
curl -H "Authorization: Bearer $ADMIN_TOKEN" http://localhost:8080/debug/routes
```

## 路由配置

路由配置文件：`docker/Routers.json`（Docker 环境）或 `pkg/config/Routers.json`（本地运行），支持热更新。

```json
[
  {
    "name": "user-service",
    "prefix": "/user",
    "targets": ["http://user-a:9001", "http://user-b:9001"],
    "require_auth": true,
    "allowed_roles": ["student", "admin"],
    "rate_limit": 5,
    "burst": 10,
    "add_request_headers": { "X-Gateway-Version": "1.0" },
    "add_response_headers": { "X-Powered-By": "BitterGate" }
  },
  {
    "name": "order-service",
    "prefix": "/order",
    "targets": ["http://order:9002"],
    "require_auth": true,
    "allowed_roles": ["admin"],
    "rate_limit": 3,
    "burst": 6
  }
]
```

## 项目结构

```
BitterGate/
├── main.go                    # 启动入口、登录接口、中间件链、metrics 独立端口
├── Dockerfile                 # 多阶段构建
├── docker-compose.yml         # 服务编排（网关 + 上游 + Prometheus + Grafana）
├── pkg/
│   ├── auth/                  # JWT 生成与校验、bcrypt 用户认证
│   ├── breaker/               # 三态熔断器
│   ├── config/                # 路由配置加载与热更新
│   ├── gateway/               # 网关核心：路由分发、限流、健康检查、指标、头部改写
│   ├── limiter/               # 令牌桶限流
│   ├── middleware/            # Trace ID、结构化日志中间件
│   ├── proxy/                 # 反向代理
│   └── router/                # 前缀树路由
├── docker/
│   ├── Routers.json           # Docker 环境路由配置
│   ├── prometheus.yml         # Prometheus 抓取配置
│   └── grafana/               # Grafana 数据源、面板、provisioning 配置
└── tools/
    ├── mock_upstream.py       # 模拟上游服务
    └── load_test.sh           # 持续压测脚本
```
