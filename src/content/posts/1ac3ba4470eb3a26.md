# Unity IL2CPP 游戏注入全记录：从 Dump 到主线程 Hook

> 目标：对 Phasmophobia 的 `GhostAI` 类编写内部注入 DLL，实现强制猎杀、显形、变速等功能。
> 核心挑战：Il2Cpp 线程安全、Photon 房主权限、方法签名验证。

---

## 一、Dump 符号

使用 [Il2CppDumper](https://github.com/Perfare/Il2CppDumper) 对 `GameAssembly.dll` + `global-metadata.dat` 做 dump。

```bash
Il2CppDumper.exe GameAssembly.dll global-metadata.dat ./output
```

输出关键文件：

| 文件 | 用途 |
|---|---|
| `dump.cs` | C# 类/方法/字段声明（含偏移量） |
| `script.json` | 函数地址映射，供 IDA/Ghidra 导入 |

从 `dump.cs` 中定位到 `GhostAI` 类，拿到 24 个行为状态枚举和完整的字段偏移表。

**⚠️ 注意：dump.cs 的反混淆器会猜错参数类型。** 比如 `ChangeState` 的第三个参数被标为 `bool`，实际是 `void*`（PhotonObjectInteract 数组指针）。后面会详述这个坑带来的后果。

## 二、Il2Cpp 运行时基础

### 对象布局

每个 Il2Cpp 托管对象都有一个 16 字节的 GC 头：

```cpp
struct Il2CppObject {
    void* klass;    // 0x00 — 类型元数据
    void* monitor;  // 0x08 — 同步块
};
```

`dump.cs` 中的字段偏移量从对象起始位置（含 GC 头）计算。例如 `GhostAI.currentState` 在 `0x30`。

### 方法解析

不硬编码 RVA，而是通过 Il2Cpp 元数据 API 运行动态解析：

```
assembly → image → class → MethodInfo → methodPointer
```

```cpp
void* dom = il2cpp_domain_get();
void* asm_ = il2cpp_domain_assembly_open(dom, "Assembly-CSharp");
void* img = il2cpp_assembly_get_image(asm_);
void* klass = il2cpp_class_from_name(img, "", "GhostAI");
MethodInfo* mi = il2cpp_class_get_method_from_name(klass, "ChangeState", 3);
uintptr_t fn = mi->methodPointer;  // 原生函数指针
```

`argsCount` 是 **C# 托管参数个数**（不含 this）。原生签名末尾固定跟一个 `MethodInfo*`。

### 字符串：只用 UTF-16

```cpp
// ❌ 错误 — il2cpp_string_new 创建的字符串在类型系统中无法匹配
Il2CppString* s = il2cpp_string_new("Ghost");

// ✅ 正确 — 手动转 UTF-16 后创建
int len = MultiByteToWideChar(CP_UTF8, 0, "Ghost", -1, nullptr, 0);
std::wstring w(len, 0);
MultiByteToWideChar(CP_UTF8, 0, "Ghost", -1, &w[0], len);
Il2CppString* s = il2cpp_string_new_utf16(w.c_str(), len - 1);
```

**这个坑花了 4 次迭代才定位到。** 现象是 `Type.GetType("GhostAI")` 在 `R/W=0x0` 处访问违规崩溃，因为字符串在 GC 堆上的内部结构与类型查找的哈希表键不匹配。

## 四、致命问题：线程安全

### 问题 1：后台线程直接调 Unity API → 崩溃

最初的方案：Input 线程直接调用 `FindGameObjectsWithTag` / `GetComponent` / `ChangeState`。

```
[CRASH] RIP=0x7FF8219AC723  R/W=0x0
```

**根因：** `CreateThread` 创建的后台线程未调用 `il2cpp_thread_attach(domain)`。

**修复：** 在每个新线程入口调用 attach。

```cpp
static DWORD WINAPI Input(LPVOID) {
    il2cpp_thread_attach(il2cpp_domain_get());  // ← 必须！
    // ...
}
```

### 问题 2：后台线程调 Photon RPC → 死锁/崩溃

修复 problem 1 后，大部分 API 通了。但 `Hunting()`（标记为 `[PunRPC]`）和 `ChangeState()` 从后台线程调用时直接冻结游戏。

**根因：** Photon 的网络同步方法内部依赖主线程的消息泵和网络回调。从后台线程调用会进入无限等待。

**解决方案：Hook `Update()` 获得主线程执行上下文。**

## 五、无代码修改的 Hook：MethodInfo 指针替换

传统 detour（覆盖函数开头字节）有两大问题：
1. 手写 trampoline 不处理 RIP-relative 指令 → 崩溃
2. Microsoft Detours 的 .lib 是 MSVC 编译的，g++ 不兼容

**最终方案：直接把 Il2Cpp 方法表里的函数指针换成我们的。**

```cpp
// 拿到指向 MethodInfo->methodPointer 的指针
uintptr_t* pp = ResolveMethodPtrPtr("Assembly-CSharp","","GhostAI","Update",0);

// 保存原始值
g_OrigUpdate = *pp;

// 替换
DWORD old;
VirtualProtect(pp, 8, PAGE_READWRITE, &old);
*pp = (uintptr_t)HookedUpdate;
VirtualProtect(pp, 8, old, &old);
```

原理：Unity 每帧调用 `Update()` 时，是从 `MethodInfo` 表读取函数指针后 `call`。替换表里的指针后，所有对 `Update` 的调用自然流入我们的 `HookedUpdate`，运行在**调用方的线程（Unity 主线程）**上。

### 命令队列模式

输入线程只负责设置标志位，主线程 Hook 负责执行：

```cpp
// Input 线程（后台）
if (GetAsyncKeyState(VK_F3)) g_Cmd = CMD_HUNT;

// HookedUpdate（主线程）
static void HookedUpdate(void* _this, void* mi) {
    switch (g_Cmd) {
    case CMD_HUNT:
        // 主线程安全！
        ChangeState(_this, 2);
        break;
    }
    g_Cmd = CMD_NONE;
    ((UpdateFn)g_OrigUpdate)(_this, mi);  // 调用原始 Update
}
```

## 六、致命问题：dump.cs 类型反混淆错误

![dnSpy 反编译对比](/img/dnSpy_iBRXsDtPzV.webp)

![IDA 方法表验证](/img/ida_OJzdEdMxdE.webp)

### 现象

`ChangeState(Wander/Idle)` 调用崩溃，尽管在同一主线程上执行。

### 分析

dump.cs 显示：
```csharp
public void ChangeState(GhostState, PhotonObjectInteract, bool force = False) { }
```

于是我们写了：
```cpp
typedef void (*ChgFn)(void*, int, void*, bool, void*);
//                         this, state, interact, force, methodInfo ← 错误！
```

但实际 Il2Cpp 方法表里存的是：
```cpp
typedef void (*ChgFn)(void*, int, void*, void*, void*);
//                         this, state, interact, void* interactArray, methodInfo
```

第三个参数是 `void*`（PhotonObjectInteract 数组），不是 `bool`。传 `false`（单字节 `0x00`）替代 `void*`（8 字节 `nullptr`）导致栈/寄存器对齐错误。

### 解决方案

**永远不要单信 dump.cs 的参数类型。** 必须参考可用的已有 SDK（如 PhasmoCheatV 的开源代码）来交叉验证函数签名。

## 七、关键问题：Photon 房主权限

### 现象

`ChangeState(Hunting)` 从主线程调用时游戏冻结（未响应）。

### 根因

状态同步方法需要通过 Photon Network 广播给所有客户端。非房主客户端无权发起状态变更，方法内部进入阻塞等待 Photon 确认，永远等不到。

### 解决方案

所有 Photon 方法调用前检查 `IsMasterClient`：

```cpp
bool IsHost() {
    return PhotonNetwork_get_IsMasterClient(nullptr);
}

case CMD_HUNT:
    if (!IsHost()) {
        Log("[-] Must be HOST to force hunt!");
    } else {
        ChangeState(_this, 2);  // 房主安全
    }
    break;
```

非房主时用字段直写作为次选方案（绕过 Photon 同步，仅本地生效）：

```cpp
case CMD_WANDER:
    if (!IsHost()) {
        *(int*)((uintptr_t)_this + 0x30) = 1;  // 字段直写
    } else {
        ChangeState(_this, 1);  // Photon 路径
    }
    break;
```

## 八、完整代码

```cpp
#include <Windows.h>
#include <cstdio>
#include <cstdarg>
#include <cstdint>
#include <string>

// ===== Console =====
static void Log(const char* f, ...) {
    va_list a; va_start(a, f); vfprintf(stdout, f, a); fflush(stdout); va_end(a);
}

// ===== VEH 崩溃调试 =====
static LONG WINAPI Crash(EXCEPTION_POINTERS* e) {
    if (e->ExceptionRecord->ExceptionCode == 0xC0000005)
        Log("[CRASH] RIP=0x%llX R/W=0x%llX\n",
            e->ContextRecord->Rip, e->ExceptionRecord->ExceptionInformation[1]);
    return EXCEPTION_CONTINUE_SEARCH;
}

// ===== Il2Cpp API 函数指针 =====
typedef void* (*t_d)(void);
typedef void  (*t_ta)(void*);
typedef void* (*t_dao)(void*, const char*);
typedef void* (*t_agi)(void*);
typedef void* (*t_cfn)(void*, const char*, const char*);
typedef void* (*t_gm)(void*, const char*, int);
typedef void* (*t_s16)(const wchar_t*, int);

static HMODULE hGA;
static t_d   il_dom;
static t_ta  il_attach;
static t_dao il_dom_asm;
static t_agi il_asm_img;
static t_cfn il_cfn;
static t_gm  il_gm;
static t_s16 il_str16;

static bool Init() {
    hGA = GetModuleHandleW(L"GameAssembly.dll");
    if (!hGA) return false;
    auto G = [](const char* n){ return GetProcAddress(hGA, n); };
    il_dom     = (t_d)  G("il2cpp_domain_get");
    il_attach  = (t_ta) G("il2cpp_thread_attach");
    il_dom_asm = (t_dao)G("il2cpp_domain_assembly_open");
    il_asm_img = (t_agi)G("il2cpp_assembly_get_image");
    il_cfn     = (t_cfn)G("il2cpp_class_from_name");
    il_gm      = (t_gm) G("il2cpp_class_get_method_from_name");
    il_str16   = (t_s16)G("il2cpp_string_new_utf16");
    if (!il_dom || !il_str16 || !il_gm) return false;
    il_attach(il_dom());
    return true;
}

// ===== Unity 字符串（UTF-16） =====
static void* NewStr(const char* s) {
    int n = MultiByteToWideChar(CP_UTF8, 0, s, -1, nullptr, 0);
    if (n <= 0) return nullptr;
    std::wstring w(n, 0);
    MultiByteToWideChar(CP_UTF8, 0, s, -1, &w[0], n);
    return il_str16(w.c_str(), n - 1);
}

// ===== 方法指针解析（返回 &MethodInfo->methodPointer） =====
static uintptr_t* ResolvePP(const char* asm_, const char* ns,
                             const char* cls, const char* m, int argc) {
    void* dom = il_dom();
    void* a = il_dom_asm(dom, asm_);
    if (!a) return nullptr;
    void* img = il_asm_img(a);
    if (!img) return nullptr;
    void* k = il_cfn(img, ns, cls);
    if (!k) return nullptr;
    return (uintptr_t*)il_gm(k, m, argc);
}

// ===== 全局状态 =====
static void* g_AI = nullptr;
static volatile int g_Cmd = 0;
static uintptr_t g_OrigUpdate  = 0;
static uintptr_t g_fn_Appear   = 0;
static uintptr_t g_fn_UnAppear = 0;
static uintptr_t g_fn_Change   = 0;
static uintptr_t g_fn_IsMaster = 0;

static bool IsHost() {
    return g_fn_IsMaster && ((bool(*)(void*))g_fn_IsMaster)(nullptr);
}

// ===== Update Hook（主线程安全） =====
static void HookedUpdate(void* _this, void* mi) {
    if (!g_AI) { g_AI = _this; Log("[!] GhostAI=%p\n", _this); }

    switch (g_Cmd) {
    case 1: {  // Print state
        int s = *(int*)((uintptr_t)_this + 0x30);
        Log("[State] %d | info=%p model=%p target=%p\n",
            s, *(void**)((uintptr_t)_this+0x38),
            *(void**)((uintptr_t)_this+0x60),
            *(void**)((uintptr_t)_this+0x110));
        break;
    }
    case 2:  // Appear
        ((void(*)(void*,int,void*))g_fn_Appear)(_this, -1, nullptr);
        Log("[*] Appear\n");
        break;
    case 3:  // Hunt (Host only)
        if (IsHost()) {
            ((void(*)(void*,int,void*,void*,void*))g_fn_Change)(_this,2,nullptr,nullptr,nullptr);
            Log("[*] Hunting\n");
        } else Log("[-] Host only\n");
        break;
    case 4:  // Wander
        if (IsHost())
            ((void(*)(void*,int,void*,void*,void*))g_fn_Change)(_this,1,nullptr,nullptr,nullptr);
        else *(int*)((uintptr_t)_this+0x30) = 1;
        Log("[*] Wander\n");
        break;
    case 5:  // Speed x10
        *(float*)((uintptr_t)_this+0xC8) = 10.f;
        *(float*)((uintptr_t)_this+0xCC) = 10.f;
        *(float*)((uintptr_t)_this+0xD0) = 10.f;
        Log("[*] Speed 10x\n");
        break;
    case 6:  // Idle
        if (IsHost())
            ((void(*)(void*,int,void*,void*,void*))g_fn_Change)(_this,0,nullptr,nullptr,nullptr);
        else *(int*)((uintptr_t)_this+0x30) = 0;
        Log("[*] Idle\n");
        break;
    }
    g_Cmd = 0;
    ((void(*)(void*,void*))g_OrigUpdate)(_this, mi);
}

// ===== 安装 Hook（MethodInfo 指针替换） =====
static bool Install() {
    uintptr_t* pp = ResolvePP("Assembly-CSharp","","GhostAI","Update",0);
    if (!pp) return false;

    g_OrigUpdate = *pp;
    DWORD old;
    VirtualProtect(pp, 8, PAGE_READWRITE, &old);
    *pp = (uintptr_t)HookedUpdate;
    VirtualProtect(pp, 8, old, &old);
    Log("[+] Hooked Update: %p -> %p\n", (void*)g_OrigUpdate, HookedUpdate);

    auto R = [](uintptr_t& out, const char* m, int c) {
        auto* p = ResolvePP("Assembly-CSharp","","GhostAI",m,c);
        out = p ? *p : 0;
    };
    R(g_fn_Change, "ChangeState", 3);
    R(g_fn_Appear, "Appear", 1);
    R(g_fn_UnAppear, "UnAppear", 0);
    { auto* p = ResolvePP("PhotonUnityNetworking","Photon.Pun","PhotonNetwork","get_IsMasterClient",0);
      g_fn_IsMaster = p ? *p : 0; }
    return true;
}

// ===== 输入线程 =====
static bool g_Run = true;
static DWORD WINAPI Input(LPVOID) {
    il_attach(il_dom());
    Log("[*] F1=State F2=Appear F3=Hunt F4=Wander F5=Speed10 F6=Idle\n");
    bool k[10]={};
    for (;g_Run;Sleep(30)) {
        auto P=[&](int vk,int n,int c){
            if(GetAsyncKeyState(vk)&0x8000){if(!k[n]){k[n]=1;g_Cmd=c;}}else k[n]=0;
        };
        P(VK_F1,1,1); P(VK_F2,2,2); P(VK_F3,3,3);
        P(VK_F4,4,4); P(VK_F5,5,5); P(VK_F6,6,6);
    }
    return 0;
}

// ===== DllMain =====
BOOL WINAPI DllMain(HINSTANCE, DWORD r, LPVOID) {
    if (r != DLL_PROCESS_ATTACH) return TRUE;
    AddVectoredExceptionHandler(1, Crash);
    AllocConsole();
    FILE* d; freopen_s(&d, "CONOUT$", "w", stdout);

    while (!GetModuleHandleW(L"GameAssembly.dll")) Sleep(500);
    Init();
    Install();
    Log("[+] Ready\n");

    HANDLE t = CreateThread(0,0,Input,0,0,0);
    if (t) CloseHandle(t);
    return TRUE;
}
```

编译：
```bash
g++ -shared -o GhostAI_Demo.dll GhostAI_Demo.cpp -std=c++17 -O2 -static
```

## 九、踩坑清单

| # | 问题 | 症状 | 根因 | 解决方案 |
|:-:|---|---|---|---|
| 1 | 字符串类型不匹配 | `Type.GetType` 崩溃，`R/W=0x0` | `il2cpp_string_new` vs `_utf16` | 只用 `il2cpp_string_new_utf16` |
| 2 | 线程未 attach | 同上崩溃点 | 新线程未注册到 Il2Cpp GC | 线程入口调 `il2cpp_thread_attach` |
| 3 | 后台线程调 Photon | 游戏冻结 | Photon 内部等待主线程回调 | Hook Update 获得主线程上下文 |
| 4 | dump.cs 类型反混淆错误 | ChangeState 崩溃 | 第三个参数是 `void*` 不是 `bool` | 交叉验证已有 SDK 的函数签名 |
| 5 | 手写 detour 不处理 RIP-relative | 游戏启动崩溃 | 覆盖代码字节时破坏指令 | 改用 MethodInfo 指针替换 |
| 6 | MSVC detours.lib + g++ 不兼容 | 链接错误 | 不同编译器的 C++ ABI | MethodInfo 替换方案跨编译器 |
| 7 | 非房主调 ChangeState | 游戏冻结/闪退 | Photon 等待同步确认超时 | 调前检查 `IsMasterClient` |
| 8 | runtime_invoke 参数格式错误 | 传 string 崩，`R/W=0x0` | 需要 `&value` 而非 `value` | 或直接用原生方法指针 |
| 9 | 字段直写被覆盖 | Hunt 状态立即被重置 | Update() 检查条件不满足 | ChangeState（房主）走完整 Photon 路径 |
| 10 | 自然猎杀中调 Stop 冻结 | 猎杀中 ChangeState 死锁 | Photon 网络冲突 | 猎杀中用字段直写，非猎杀走 ChangeState |

## 十、总结

这个项目最核心的三条经验：

1. **线程安全是第一优先级** — IL2CPP 游戏的托管对象不能跨线程随意访问。`il2cpp_thread_attach` 是必要但不充分的——Photon 网络方法还要求主线程。

2. **dump.cs 不能全信** — 反混淆器会猜错参数类型。务必用已有开源项目（如 PhasmoCheatV）的 SDK 定义交叉验证。

3. **MethodInfo 指针替换是最干净的 Il2Cpp Hook 方式** — 零代码修改，跨编译器兼容，不依赖第三方库。唯一局限是只能 Hook Il2Cpp 托管方法。
