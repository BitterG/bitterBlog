# frida安装&命令行使用

## 一、安装

手机端

github下载对应<https://github.com/frida/frida>

使用adb devices查看设备是否连接

使用adb发送到手机/data/local/tmp/下修改权限后启动

主机端

使用pip3安装

```plain
pip3 install frida==16.7.19
pip3 install firda-tools==12.5.1
```

#### attach模式

```plain
frida -U -n 应用名/包名 -l 脚本.js
  -U usb连接
  -n 附加到指定名称的进程（较新版本attach模式使用应用名）
  -l 加载指定脚本
```

#### spawn模式

```plain
frida -Uf 包名 -l 脚本.js --pause
  -U usb连接
  -f 启动目标文件（spawn模式使用包名）
  -l 加载指定脚本
  --pause 启动后让主线程暂停
```

Spawned `xxx.xxx.xxx`. Use %resume to let the main thread start executing!

之后根据提示输入%resume继续进程

#### 16.7.19 frida --help中文版

```plain
用法: frida [选项] 目标

位置参数:
  args                  额外参数和/或目标

选项:
  -h, --help            显示此帮助信息并退出
  -D ID, --device ID    连接到指定ID的设备
  -U, --usb             连接到USB设备
  -R, --remote          连接到远程frida-server
  -H HOST, --host HOST  连接到指定HOST的远程frida-server
  --certificate CERTIFICATE
                        与HOST进行TLS通信，期望CERTIFICATE证书
  --origin ORIGIN       连接远程服务器时设置“Origin”头为ORIGIN
  --token TOKEN         使用TOKEN认证HOST
  --keepalive-interval INTERVAL
                        设置保活间隔（秒），0为禁用（默认-1自动选择）
  --p2p                 与目标建立点对点连接
  --stun-server ADDRESS
                        设置用于--p2p的STUN服务器地址
  --relay address,username,password,turn-{udp,tcp,tls}
                        添加用于--p2p的中继
  -f TARGET, --file TARGET
                        启动（spawn）目标文件
  -F, --attach-frontmost
                        附加到当前前台应用
  -n NAME, --attach-name NAME
                        附加到指定名称的进程
  -N IDENTIFIER, --attach-identifier IDENTIFIER
                        附加到指定标识符的进程
  -p PID, --attach-pid PID
                        附加到指定PID的进程
  -W PATTERN, --await PATTERN
                        等待匹配PATTERN的spawn事件
  --stdio {inherit,pipe}
                        启动时的stdio行为（默认“inherit”）
  --aux option          启动时设置aux选项，如“uid=(int)42”（支持类型：string, bool, int）
  --realm {native,emulated}
                        附加的realm类型
  --runtime {qjs,v8}    使用的脚本运行时
  --debug               启用Node.js兼容的脚本调试器
  --squelch-crash       启用后，崩溃时不会在控制台输出报告
  -O FILE, --options-file FILE
                        包含额外命令行选项的文本文件
  --version             显示程序版本号并退出
  -l SCRIPT, --load SCRIPT
                        加载脚本
  -P PARAMETERS_JSON, --parameters PARAMETERS_JSON
                        以JSON格式传递参数，类似Gadget
  -C USER_CMODULE, --cmodule USER_CMODULE
                        加载CMODULE
  --toolchain {any,internal,external}
                        编译CModule时使用的工具链
  -c CODESHARE_URI, --codeshare CODESHARE_URI
                        加载CODESHARE_URI
  -e CODE, --eval CODE  执行代码
  -q                    静默模式（无交互），加载-l和-e后退出
  -t TIMEOUT, --timeout TIMEOUT
                        静默模式下终止前等待的秒数
  --pause               启动后让主线程暂停
  -o LOGFILE, --output LOGFILE
                        输出到日志文件
  --eternalize          在退出前让脚本常驻
  --exit-on-error       脚本遇到异常时以1退出
  --kill-on-exit        Frida退出时杀死spawn的程序
  --auto-perform        自动用Java.perform包裹输入的代码
  --auto-reload         启用脚本和c模块的自动重载（默认开启，未来将强制要求）
  --no-auto-reload      禁用脚本和c模块的自动重载
```
