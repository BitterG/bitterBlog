# Google Protocol实现对象的传递

#### 1.什么是Protocol？
<font style="color:rgb(51, 51, 51);">protocol buffer是由google推出一种数据编码格式，不依赖平台和语言，类似于xml和json</font>

#### 2.为什么用？
<font style="color:rgb(51, 51, 51);">和json或者xml相比，protocol buffer的解析速度更快，编码后的字节数更少。</font>

<font style="color:rgb(51, 51, 51);">其中解析速度的相关比较可以参看相关文章，这并不是本系列关心的重点，而</font>**<font style="color:rgb(51, 51, 51);">字节数的减少</font>**<font style="color:rgb(51, 51, 51);">将会是后续扩展和优化的重点。</font>

<font style="color:rgb(51, 51, 51);">另外，比json和xml更便利的是，开发者只需要编写一份.proto的描述文件，就可以通过google提供的编译器生成不同平台的模型代码，包括java、C#等等，而不需要手动进行模型编写。</font>

### 3.服务器端实现
加入pipeline中加入 ProtobufDecoder 并指定传递的对象类型

```java
package netty.codec2;

import io.netty.bootstrap.ServerBootstrap;
import io.netty.channel.*;
import io.netty.channel.nio.NioEventLoopGroup;
import io.netty.channel.socket.SocketChannel;
import io.netty.channel.socket.nio.NioServerSocketChannel;
import io.netty.handler.codec.protobuf.ProtobufDecoder;
import netty.codec.StudentPojo;

public class NettyServer {
    public static void main(String[] args) throws InterruptedException {

        //1.创建BossGroup 和 WorkerGroup, 是两个线程组
        //2.boosGroup只处理连接请求，业务处理交给workerGroup
        //3.两个都是无限循环
        EventLoopGroup boosGroup = new NioEventLoopGroup();

        EventLoopGroup workerGroup = new NioEventLoopGroup();

        try {
            //创建服务器端启动对象，配置参数
            ServerBootstrap bootstrap = new ServerBootstrap();
            bootstrap.group(boosGroup, workerGroup) //设置两个线程组
                    .channel(NioServerSocketChannel.class) //使用NioServerSocketChannel作为服务器通道实现(bossGroup)
                    .option(ChannelOption.SO_BACKLOG, 128) //设置线程队列
                    .childOption(ChannelOption.SO_KEEPALIVE, true) //设置保持活动连接状态
                    .childHandler(new ChannelInitializer<SocketChannel>() { //创建通道初始化对象（匿名方式）
                        @Override
                        protected void initChannel(SocketChannel socketChannel) throws Exception {
                            ChannelPipeline pipeline = socketChannel.pipeline();
                            //加入protoBufDecoder
                            //需指定对哪种类型进行解码
                            pipeline.addLast("decoder", new ProtobufDecoder(MyDataInfo.MyMessage.getDefaultInstance()));
                            pipeline.addLast(new NettyServerHandler());
                        }
                    }); //给WorkerGroup的EventLoop设置管道处理器（重要）

            System.out.println("服务器准备完成");

            //绑定端口并同步, 生成 ChannelFuture 对象
            ChannelFuture cf = bootstrap.bind(6668).sync();

            //Future Listener 机制
            //给cf注册监听器，监控关心的事件
            cf.addListener(new ChannelFutureListener() {
                @Override
                public void operationComplete(ChannelFuture channelFuture) throws Exception {
                    if (channelFuture.isSuccess()) {
                        System.out.println("监听端口6668 成功！");
                    } else {
                        System.out.println("监听端口6668 失败！");
                    }
                }
            });

            //对关闭通道进行 监听
            cf.channel().closeFuture().sync();
        } finally {
            //关闭
            boosGroup.shutdownGracefully();
            workerGroup.shutdownGracefully();
        }
    }
}

```

### 4.客户端实现
可以不是java写的而客户端（这里使用java客户端演示）

```java
package netty.codec2;

import io.netty.bootstrap.Bootstrap;
import io.netty.channel.ChannelFuture;
import io.netty.channel.ChannelInitializer;
import io.netty.channel.ChannelPipeline;
import io.netty.channel.nio.NioEventLoopGroup;
import io.netty.channel.socket.SocketChannel;
import io.netty.channel.socket.nio.NioSocketChannel;
import io.netty.handler.codec.protobuf.ProtobufEncoder;

public class NettyClinet {
    public static void main(String[] args) throws InterruptedException {
        //客户端需要一个事件循环组
        NioEventLoopGroup eventLoopGroup = new NioEventLoopGroup();

        try {
            //创建启动对象 不是ServerBootstrap 是 BootStrap
            Bootstrap bootstrap = new Bootstrap();
            //设置相关参数
            bootstrap.group(eventLoopGroup) //设置线程组
                    .channel(NioSocketChannel.class) //设置客户端通道的实现类
                    .handler(new ChannelInitializer<SocketChannel>() { //初始化channel使用SocketChannel
                        @Override
                        protected void initChannel(SocketChannel socketChannel) throws Exception {
                            ChannelPipeline pipeline = socketChannel.pipeline();
                            pipeline.addLast("encoder", new ProtobufEncoder());
                            pipeline.addLast(new NettyClientHandler()); //加入自己的处理器
                        }
                    });

            System.out.println("客户端准备完毕");

            // 连接服务器
            // ChannelFuture 设计Netty的异步模型
            ChannelFuture channelFuture = bootstrap.connect("127.0.0.1", 6668).sync();

            // 为关闭通道增加监听
            channelFuture.channel().closeFuture().sync();
        } finally {
            //关闭
            eventLoopGroup.shutdownGracefully();
        }
    }
}

```

### 6.Client的Handler实现
创建proto转换的类型的对象并设置值，之后发送给服务器

Demo中随机创建两种中的一种发送给服务端

```java
package netty.codec2;

import io.netty.buffer.ByteBuf;
import io.netty.channel.ChannelHandlerContext;
import io.netty.channel.ChannelInboundHandlerAdapter;

import java.nio.charset.StandardCharsets;
import java.util.Random;

public class NettyClientHandler extends ChannelInboundHandlerAdapter {

    //当通道就绪时，会触发该方法
    @Override
    public void channelActive(ChannelHandlerContext ctx) throws Exception {
        //随机发送Student或者Worker
        int i = new Random().nextInt(3);
        // TODO
        MyDataInfo.MyMessage message = null;

        if (0 == i) {
            //发送学生
            message = MyDataInfo.MyMessage
            .newBuilder()
            .setDataType(MyDataInfo.MyMessage.DataType.StudentType)
            .setStudent(
                MyDataInfo.Student.newBuilder().setName("张三").setId(20).build()
            ).build();
        } else {
            //发送worker对象
            message = MyDataInfo.MyMessage
            .newBuilder()
            .setDataType(MyDataInfo.MyMessage.DataType.WorkerType)
            .setWorker(
                MyDataInfo.Worker.newBuilder().setName("李四").setAge(28).build()
            ).build();
        }

        ctx.writeAndFlush(message);
    }

    //    有数据可读时触发
    @Override
    public void channelRead(ChannelHandlerContext ctx, Object msg) throws Exception {
        //转成netty的buf
        ByteBuf buf = (ByteBuf) msg;

        System.out.println("服务器回复的信息：" + buf.toString(StandardCharsets.UTF_8));
        System.out.println("服务器地址：" + ctx.channel().remoteAddress());
    }

    //异常处理
    @Override
    public void exceptionCaught(ChannelHandlerContext ctx, Throwable cause) throws Exception {
        cause.printStackTrace();
        ctx.close();
    }

}
```

### 6.protoBuf文件
```protobuf
syntax = "proto3"; //标识proto版本
option optimize_for = SPEED; //快速解析
option java_package = "netty.codec2"; //指定生成到哪个包下
option java_outer_classname = "MyDataInfo"; //外部类名称

//protoBuf可以使用message管理其他message
message MyMessage {
  //定义枚举类型
  enum DataType {
    StudentType = 0; //3版本要求enum编号要求从零开始
    WorkerType = 1;
}

  //用data_type标识传的是哪个枚举类型
  DataType data_type = 1;
  //表示每次枚举类型最多只能出现一个（Student或Worker），节省空间
  oneof dataBody {
    Student student = 2;
    Worker worker = 3;
}
}

message Student {
  int32 id = 1; //属性，编号1
  string name = 2;
}
message Worker {
  string name = 1;
  int32 age = 2;
}
```

### 7.将proto文件解析成对应语言的对象文件
1.导入依赖（版本需要和解析器的版本一致）

例如：protoc-25.1-win64 对相应 <version>3.25.1</version>

```xml
<dependency>
  <groupId>com.google.protobuf</groupId>
  <artifactId>protobuf-java</artifactId>
  <version>3.25.1</version>
</dependency>
```

2.将proto文件放入\protoc-25.1-win64\bin\ (里面有一个 proto.xxx 解析启动文件)

使用命令行执行

:::color1
>protoc.exe --java_out=. XXX.proto

:::

可编辑proto文件中的option java_package = "路径地址" 指定生成的文件输出在什么位置
