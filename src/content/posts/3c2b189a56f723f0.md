# I/O的概念

<font style="color:rgb(18, 18, 18);">java编程中，所有 I/O 都被视为单个的字节的移动，通过一个称为 Stream 的对象一次移动一个字节。流 I/O 用于与外部世界接触。它也在内部使用，用于将对象转换为字节，然后再转换回对象。</font>

<font style="color:rgb(18, 18, 18);"></font>

# <font style="color:rgb(18, 18, 18);">NIO的优势/为什么用NIO</font>
<font style="color:rgb(18, 18, 18);">NIO 与原来的 I/O 有同样的作用和目的</font>

<font style="color:#000000;">NIO使用的</font><font style="color:#1DC0C9;"> 块 I/O </font><font style="color:rgb(18, 18, 18);">的效率可以比</font><font style="color:#1DC0C9;">流 I/O</font><font style="color:rgb(18, 18, 18);"> 高许多。</font>

<font style="color:rgb(18, 18, 18);"></font>

<font style="color:rgb(18, 18, 18);">NIO 将最耗时的 I/O 操作(即填充和提取缓冲区)转移回操作系统，因而可以极大地提高速度。</font>

# <font style="color:rgb(18, 18, 18);">流IO与快IO的比较</font>
<font style="color:rgb(18, 18, 18);">原来的 I/O 库(在 java.io.*中) 与 NIO 最重要的区别是数据打包和传输的方式。正如前面提到的，原来的 I/O 以流的方式处理数据，而 NIO 以块的方式处理数据。</font>

<font style="color:rgb(18, 18, 18);"></font>

<font style="color:rgb(18, 18, 18);">面向流 的 I/O 系统一次一个字节地处理数据。一个输入流产生一个字节的数据，一个输出流消费一个字节的数据。为流式数据创建过滤器非常容易。链接几个过滤器，以便每个过滤器只负责单个复杂处理机制的一部分，这样也是相对简单的。不利的一面是，面向流的 I/O 通常相当慢。</font>

<font style="color:rgb(18, 18, 18);">一个 面向块 的 I/O 系统以块的形式处理数据。每一个操作都在一步中产生或者消费一个数据块。按块处理数据比按(流式的)字节处理数据要快得多。但是面向块的 I/O 缺少一些面向流的 I/O 所具有的优雅性和简单性。</font>
