## 背景

[PhasmoCheatV](https://github.com/ViniLog789/PhasmoCheatV-Recode) 是一款 Phasmophobia（恐鬼症）的辅助工具，支持英文、俄文和中文三种语言。原项目的中文翻译覆盖率不到 80%，有大量 key 在国际化系统中缺失中文字段，导致切换中文后部分界面显示 `China not supported..##KEY_NAME`。

我用了一个周末把全部翻译补齐，并顺手修了几个国际化架构上的 bug。

## 工作量

- 新增翻译 key：**120+ 个**
- 修改文件：**10 个**
- 涉及行数：约 +300 / -100

## 一、补全缺失的中文翻译

项目用的国际化系统很简单：一个 `ADD_STR(key, en, ru, zh)` 宏，在 `Translations.h` 中集中注册所有翻译。问题出在两处：

### 覆盖性缺失

文件末尾 500+ 行的 `_Header` keys 是为功能分类面板定义的标题，但只有英文和俄文，中文参数留空。切换中文后这些标题全部显示为 `China not supported..##xxx_Header`。

### 结构性重复

有 3 个 key（`Evidence`、`GhostType`、`GhostAge`）被定义了两次：第一次在各自功能模块中有完整四语翻译，第二次在 GhostPanel 模块中却只有三语（缺中文）。由于 `ADD_STR` 采用覆盖策略，后定义的会覆盖先定义的，导致中文被擦除。

另外，`EnableSpiritBoxAlw` 这个 key 在 `spiritboxalw.cpp` 中使用了，但从来没在 `Translations.h` 中定义过。

```cpp
// 修复前：GhostPanel 重复定义覆盖了有中文的版本
ADD_STR("Evidence", "Evidence", u8"Улики");              // 缺中文，覆盖了前面的

// 修复后：直接删除重复项，保留首次定义的完整版本
// 前面已有 ADD_STR("Evidence", "Evidence", u8"Улика", u8"证据");
```

## 二、快捷键界面国际化重写

### 问题

快捷键列表显示的功能名称在切换语言后会错乱——中文模式下显示俄语，俄语模式下出现 `?`。

### 根因

`BButton`（带快捷键绑定的按钮）在创建 bind 时将翻译后的文本嵌入到了唯一标识符中：

```cpp
// 旧代码
std::string uniqueKey = std::string(label) + "##" + bindId;
// label = LANG("ForceAppear") = "强制鬼魂显形"（中文环境下）
// uniqueKey = "强制鬼魂显形##ghostModifier_forceAppear"
```

切换语言后，`LANG("ForceAppear")` 返回了英文 `"Force appear"`，但 bind 的唯一标识符仍然是中文的。两个不同语言的 bind 同时存在，显示错乱。

### 修复

重写 `BButtonImpl`，将 LANG key 和显示文本分离：

```cpp
// 新代码
std::string uniqueKey = std::string(labelKey) + "##" + bindId;
// labelKey = "ForceAppear"（语言无关）
// uniqueKey = "ForceAppear##ghostModifier_forceAppear"

std::string displayLabel = std::string(LANG(labelKey)) + "##" + bindId;
// displayLabel = "强制鬼魂显形##ghostModifier_forceAppear"（动态翻译）
```

快捷键列表的显示逻辑也相应重写，每次渲染时根据 uniqueKey 动态查 `LANG()`，确保始终跟当前语言一致。

## 三、鬼魂类型本地化

鬼魂编辑器（Ghost Designer）的下拉框中 30 种鬼魂名称全是硬编码英文。我为每种鬼魂添加了完整的三语翻译：

```
Spirit      | Дух       | 魂魄
Wraith      | Мираж     | 魅影
Phantom     | Фантом    | 幻影
Poltergeist | Полтергейст | 骚灵
Banshee     | Банши     | 女妖
...
Kormos      | Кормос    | 盲灵
Aswang      | Асванг    | 阿斯旺
```

俄文名来源于 Phasmophobia 社区和游戏官方俄语本地化。同时发现下拉列表缺少了两个新鬼魂 `Kormos` 和 `Aswang`，一并补上。

## 五、涉及的技术点

- **IL2CPP 反射**：通过 `AddFontFromFileTTF` 加载中文字体，回退到系统微软雅黑
- **ImGui 国际化**：`ImFontConfig` + `GlyphRangesChineseFull` 加载完整 CJK 字形
- **MSBuild 配置**：`stdcpp20`、`PlatformToolset v143/v145`、`RuntimeLibrary`
- **Hook 生命周期**：`FirstPersonController_Update` → `FullbrightMain()` 每帧调用

## 总结

这次贡献从发现 bug 到完整修复，主要踩了三个坑：

1. **国际化系统的覆盖语义**：同一个 key 多次调用 `ADD_STR` 会覆盖，不是追加
2. **UI 标识符与显示文本耦合**：把翻译后的文本当 ID 用必然会出问题
3. **鬼魂类型枚举与下拉列表不同步**：SDK 枚举有 30 个值，UI 只列了 28 个

如果你也在做类似的多语言维护，建议一开始就把"标识符用常量、显示用翻译"的原则贯彻到底，避免后期重构。
