# Netty服务器的实现Demo

## 服务端实现
1.创建BoosGroup和workGroup

2.使用ServerBootstrap serverBootstrap = new ServerBootstrap();创建服务器对象

3.绑定工作线程（两个Group）

4.指定通道的实现

5.添加子Group的 初始化器	.childHandler(new 自定义初始化器() 若以匿名的方式添加如下Demo

  在pipeline里添加自定义handler，非匿名方式同理); 

（自定义Handler需要继承<font style="color:#000000;">extends SimpleChannelInboundHandler<HttpObject>并重写方法</font>）

6.bind指定服务器端口 ChannelFuture channelFuture = serverBootstrap.bind(8080).sync();

（**Future.sync()** **<font style="color:rgb(77, 77, 77);">会等待异步事件执行完成，并且返回自身</font>**）

7.可对ChannelFuture对象添加监听器俩面有方法回调（如下面的代码Demo）

8.使用try finally包裹代码 最后在finally块里关闭两个线程池

![图片](/img/netty/e6426bfca8d5bf80/1701783397093-73edecee-6897-40cf-abb6-e22e082c3d40-798359.webp)

```java
package netty.simple;

import io.netty.bootstrap.ServerBootstrap;
import io.netty.channel.*;
import io.netty.channel.nio.NioEventLoopGroup;
import io.netty.channel.socket.SocketChannel;
import io.netty.channel.socket.nio.NioServerSocketChannel;
import io.netty.channel.socket.nio.NioSocketChannel;
import nio.NioServer;

public class NettyServer {
    public static void main(String[] args) throws InterruptedException {

        //1.创建BossGroup 和 WorkerGroup, 是两个线程组
        //2.boosGroup只处理连接请求，业务处理交给workerGroup
        //3.两个都是无限循环
        EventLoopGroup boosGroup = new NioEventLoopGroup();

        EventLoopGroup workerGroup = new NioEventLoopGroup();

        try {
            //创建服务器端启动对象，配置参数
            ServerBootstrap bootstrap = new ServerBootstrap();
            bootstrap.group(boosGroup, workerGroup) //设置两个线程组
                    .channel(NioServerSocketChannel.class) //使用NioServerSocketChannel作为服务器通道实现(bossGroup)
                    .option(ChannelOption.SO_BACKLOG, 128) //设置线程队列
                    .childOption(ChannelOption.SO_KEEPALIVE, true) //设置保持活动连接状态
                    .childHandler(new ChannelInitializer<SocketChannel>() { //创建通道初始化对象（匿名方式）
                        @Override
                        protected void initChannel(SocketChannel socketChannel) throws Exception {
                            //每一个客户端socket都会经过这里
                            //这里可以管理一个socket集合，方便推送消息
                            System.out.println("客户端socketChannel" + socketChannel.hashCode());
                            socketChannel.pipeline().addLast(new NettyServerHandler());
                        }
                    }); //给WorkerGroup的EventLoop设置管道处理器（重要）

            System.out.println("服务器准备完成");

            //绑定端口并同步, 生成 ChannelFuture 对象
            ChannelFuture cf = bootstrap.bind(6668).sync();

            //Future Listener 机制
            //给cf注册监听器，监控关心的事件
            cf.addListener(new ChannelFutureListener() {
                @Override
                public void operationComplete(ChannelFuture channelFuture) throws Exception {
                    if (cf.isSuccess()) {
                        System.out.println("监听端口6668 成功！");
                    } else {
                        System.out.println("监听端口6668 失败！");
                    }
                }
            });

            //对关闭通道进行 监听
            cf.channel().closeFuture().sync();
        } finally {
            //关闭
            boosGroup.shutdownGracefully();
            workerGroup.shutdownGracefully();
        }
    }
}

```

## 客户端实现
1.同服务端创建NioEventLoopGroup线程组 无BossGroup

2.客户端使用的是Bootstrap（服务端是使用的ServerBootstrap）

3.和服务端一样需要绑定线程组.group(）

4.使用.channel(NioSocketChannel.class) //设置客户端通道的实现类

5.使用.handler()绑定初始化处理器（不需要childHandler因为没有子线程组）

5.使用.handler(new 自定义初始化器())绑定初始化处理器（不需要childHandler因为没有子线程组）(new 自定     义初始化器() 若以匿名的方式添加如下Demo 在pipeline里添加自定义handler，非匿名方式同理); 

6.bootstrap.connect().sync();连接服务器

（**Future.sync()** **<font style="color:rgb(77, 77, 77);">会等待异步事件执行完成，并且返回自身</font>**）

7.sync()会返回ChannelFuture可使用对象添加监听

8.使用try finally包裹代码 最后在finally块里关闭线程池

```java
package netty.simple;

import io.netty.bootstrap.Bootstrap;
import io.netty.channel.ChannelFuture;
import io.netty.channel.ChannelInitializer;
import io.netty.channel.nio.NioEventLoopGroup;
import io.netty.channel.socket.SocketChannel;
import io.netty.channel.socket.nio.NioSocketChannel;

public class NettyClinet {
    public static void main(String[] args) throws InterruptedException {
        //客户端需要一个事件循环组
        NioEventLoopGroup eventLoopGroup = new NioEventLoopGroup();

        try {
            //创建启动对象 不是ServerBootstrap 是 BootStrap
            Bootstrap bootstrap = new Bootstrap();
            //设置相关参数
            bootstrap.group(eventLoopGroup) //设置线程组
            .channel(NioSocketChannel.class) //设置客户端通道的实现类
            .handler(new ChannelInitializer<SocketChannel>() { //初始化channel使用SocketChannel
                @Override
                protected void initChannel(SocketChannel socketChannel) throws Exception {
                    socketChannel.pipeline().addLast(new NettyClientHandler()); //加入自己的处理器
                }
            });

            System.out.println("客户端准备完毕");

            // 连接服务器
            // ChannelFuture 设计Netty的异步模型
            ChannelFuture channelFuture = bootstrap.connect("127.0.0.1", 6668).sync();

            // 为关闭通道增加监听
            channelFuture.channel().closeFuture().sync();
        } finally {
            //关闭
            eventLoopGroup.shutdownGracefully();
        }
    }
}
```

## 自定义服务端Handler
1.创建一个类并继承ChannelInboundHandlerAdapter重写内部需要的方法

2.channelRead发生读取，channelReadComplete读取完成，exceptionCaught发生异常

3.可通过ChannelHandlerContext上下文拿到管道(pipline),通道(channel)

<font style="color:#000000;">例如ctx.channel().eventLoop().execute() 或者</font>

<font style="color:#000000;">ctx.channel().eventLoop().schedule() 提交</font><font style="color:#74B602;">普通任务</font><font style="color:#000000;">和</font><font style="color:#74B602;">定时任务</font><font style="color:#000000;">到任务队列（异步操作）</font>

```java
package netty.simple;

import io.netty.buffer.ByteBuf;
import io.netty.buffer.Unpooled;
import io.netty.channel.ChannelHandlerContext;
import io.netty.channel.ChannelInboundHandlerAdapter;
import io.netty.util.CharsetUtil;

import java.nio.ByteBuffer;
import java.util.concurrent.TimeUnit;

public class NettyServerHandler extends ChannelInboundHandlerAdapter {

    //读取实际数据
    /**
     *
     * @param ctx 上下文对象，有管道pipeline，通道channel，地址
     * @param msg 客户端发送的数据，默认Object
     * @throws Exception
     */
    @Override
    public void channelRead(ChannelHandlerContext ctx, Object msg) throws Exception {
        //        System.out.println("server ctx =" + ctx);
        //        //将msg转为byteBuf 由Netty提供的
        //        ByteBuf buf = (ByteBuf) msg;
        //        System.out.println("客户端发送的消息是：" + buf.toString(CharsetUtil.UTF_8));
        //        System.out.println("客户端地址：" + ctx.channel().remoteAddress());

        //比如有一个耗时操作业务 -> 异步执行 提交该channel 对应的
        //EventLoop 的 taskQueue中,

        //解决方案1 用户自定义的普通任务 提交到当前EventLoop的队列
        ctx.channel().eventLoop().execute(new Runnable() {
            @Override
            public void run() {
                try {
                    Thread.sleep(1000 * 10);
                    ctx.writeAndFlush(Unpooled.copiedBuffer("hello, 客户端2", CharsetUtil.UTF_8));
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        });

        //解决方案2 用户自定义"定时"任务 -> 该任务提交到 scheduleTaskQueue
        //提交到当前EventLoop的队列
        ctx.channel().eventLoop().schedule(new Runnable() {
            @Override
            public void run() {
                try {
                    Thread.sleep(1000 * 10);
                    ctx.writeAndFlush(Unpooled.copiedBuffer("hello, 客户端3", CharsetUtil.UTF_8));
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        }, 5, TimeUnit.SECONDS);
    }

    //数据读取完毕
    @Override
    public void channelReadComplete(ChannelHandlerContext ctx) throws Exception {
        // 数据写入缓存并刷新, write + flush 方法结合
        //对发送的数据进行编码返回
        ctx.writeAndFlush(Unpooled.copiedBuffer("hello, 客户端1", CharsetUtil.UTF_8));
    }

    //处理异常, 一般需要关闭通道
    @Override
    public void exceptionCaught(ChannelHandlerContext ctx, Throwable cause) throws Exception {
        ctx.close();
    }
}
```

## 自定义客户端Handler
1. 继承 extends ChannelInboundHandlerAdapter 并重写方法

2.对应的事件触发对应的方法，重写逻辑即可

```java
package netty.simple;

import io.netty.buffer.ByteBuf;
import io.netty.buffer.Unpooled;
import io.netty.channel.ChannelHandlerContext;
import io.netty.channel.ChannelInboundHandlerAdapter;

import java.nio.charset.StandardCharsets;

public class NettyClientHandler extends ChannelInboundHandlerAdapter {

    //当通道就绪时，会触发该方法
    @Override
    public void channelActive(ChannelHandlerContext ctx) throws Exception {
        System.out.println("client " + ctx);
        ctx.writeAndFlush(Unpooled.copiedBuffer("Hello,server!", StandardCharsets.UTF_8));
    }

    //有数据可读时触发
    @Override
    public void channelRead(ChannelHandlerContext ctx, Object msg) throws Exception {
        //转成netty的buf
        ByteBuf buf = (ByteBuf) msg;

        System.out.println("服务器回复的信息：" + buf.toString(StandardCharsets.UTF_8));
        System.out.println("服务器地址：" + ctx.channel().remoteAddress());
    }

    //异常处理
    @Override
    public void exceptionCaught(ChannelHandlerContext ctx, Throwable cause) throws Exception {
        cause.printStackTrace();
        ctx.close();
    }
}
```

import io.netty.channel.ChannelInitializer;
import io.netty.channel.ChannelPipeline;
import io.netty.channel.socket.SocketChannel;
import io.netty.handler.codec.http.HttpServerCodec;

public class TestServerInit extends ChannelInitializer<SocketChannel> {
    @Override
    protected void initChannel(SocketChannel socketChannel) throws Exception {
        //向管道加入处理器

        //得到管道
        ChannelPipeline pipeline = socketChannel.pipeline();
        //加入netty提供的 httpServerCodec
        //处理http编解码
        pipeline.addLast("MyHttpServerCodec", new HttpServerCodec())
        .addLast("MyHttpServerHandler", new TestHttpServerHandler());
