# ThreadLocal-为线程提供局部变量

:::color1
## ThreadLocal-为线程提供局部变量
:::

通常为类中的<font style="color:#117CEE;">私有静态</font>字段，目的是状态(例如，用户ID或事物ID)与线程关联起来。

让每个线程都有自己的本地变量副本，对外提供get，set方法，避免了线程安全问题👍

```java
class House {
    int saleCount = 0;
    public synchronized void setSaleCount() {
        saleCount++;
    }

    /* 匿名内部类写法
    ThreadLocal<Integer> saleVolume = new ThreadLocal<Integer>(){
        @Override
        protected Integer initialValue() {
        	return 0;
    	}
    };
    */

    //Lambda表达式写法 推荐
    ThreadLocal<Integer> saleVolume = ThreadLocal.withInitial(() -> 0);

    public void saleVolumeByThreadLocal() {
        saleVolume.set(saleVolume.get() + 1);
    }
}

public class ThreadLocal01 {
    public static void main(String[] args) throws InterruptedException {
        House house = new House();

        for (int i = 0; i < 5; i++) {
            new Thread(() -> {
                int num = new Random().nextInt(5) + 1;
                for (int j=0; j < num; j++) {
                    house.setSaleCount(); //总计
                    house.saleVolumeByThreadLocal();	//ThreadLocal
                }
                System.out.println(Thread.currentThread().getName() + "号线程Thread卖出为" + house.saleVolume.get());
            }, String.valueOf(i)).start();
        }
        TimeUnit.MILLISECONDS.sleep(10L); //保证执行完成
        System.out.println("总计卖出" + house.saleCount);
    }
}
```

## 及时回收自定义的ThreadLocal变量
:::

尤其在线程池下复用线程，不及时清楚可能会造成内存泄露

调用<font style="color:#1DC0C9;">remove()</font>方法进行清理，后变为初始化时设置的默认值

```java
class MyData {
    //初始化ThreadLocal 默认值设置为0 remove()后变为默认值
    ThreadLocal<Integer> threadLocalField = ThreadLocal.withInitial(() -> 0);

    public void add() {
        threadLocalField.set(threadLocalField.get() + 1);
    }
}

public class ThreadLocal02 {
    public static void main(String[] args) {
        MyData myData = new MyData();
        ExecutorService threadPool = Executors.newFixedThreadPool(3); //创建线程池

        try {
            for (int i = 0; i < 10; i++) {
                threadPool.submit(() -> {
                    try {
                        Integer before = myData.threadLocalField.get();
                        myData.add();
                        Integer after = myData.threadLocalField.get();
                        System.out.println(Thread.currentThread().getName()+ "\t" + "before:" + before + "after:" + after);
                    } finally {
                        //每次清理当前线程ThreadLocal
                        //变为默认值0
                        myData.threadLocalField.remove();  
                    }
                });
            }
        } catch (Exception e) {
            e.printStackTrace();
        } finally {
            threadPool.shutdown(); //关闭线程池
        }
    }
}
```

## Thread，ThreadLocal，ThreadLocalMap 关系
:::

ThreadLocalMap是ThreadLocal的静态内部类，ThreadLocalMap中还存在一个名为Entry的静态内部类继承了WeakReference<ThreadLocal<?>>

即：<font style="color:#1DC0C9;">ThreadLocal -> ThreadLocalMap -> Entry extends WeakReference<ThreadLocal<?>></font>

![图片](/img/juc/9c105a0ec40495c4/1674129925149-1a119638-4c68-479a-9570-13be131f2acf-071297.png) jvm内部维护了一个线程版本Map<ThreadLocal, Value>

通过ThreadLocal对象的set方法，将<font style="color:#74B602;">线程对象当作key</font>设置进去，取值时同理用当前线程去取值，

从而保证了人手只有一份，在并发下的绝对安全

:::color1
## ThreadLocal为什么是弱引用问题
:::

保证关闭线程时，原本对应的K,v键值对（key为弱引用）的强引用value被及时回收;

存在问题：若线程被指向null则还存在key为null（强引用）的键值对（null:value）无法被回收

导致堆积造成内存泄露，在多线程下影响较大

<font style="color:#1DC0C9;">调用get，set，remove方法时都会调用expungeStaleEntry方法来清理脏entry;</font>

即：寻找key=null的Entry，然后进行删除

<font style="color:#DF2A3F;">推荐</font>：使用ThreadLocal.withInitial(() -> 初始值); 的方式初始化

把ThreadLocal修饰为static 只需要初始化一次就够了实际上由ThreadLocalMap存储（static默认就是强引用不会被回收）

<font style="color:#DF2A3F;">使用完成记得手动remove()回收</font>

## 软引用
:::

软引用是相对强引用弱化一些的引用，需要java.lang.ref.SoftReference类来实现，可以让对象豁免一些垃圾收集

对于软引用对象当<font style="color:#74B602;">内存充足不会被回收，内存不足会被回收</font>，比如高速缓存中会用到软引用。

```java
class Paper {
    //gc回收之前会自动调用该方法 一般不需要自己重写
    @Override
    protected void finalize() throws Throwable {
        System.out.println("即将调用gc回收无用对象~");
    }
}

public class gcDemo {
    public static void main(String[] args) throws InterruptedException {

        //创建软引用
        SoftReference<Paper> softReference = new SoftReference<>(new Paper());
        System.gc(); //此时内存充足不会回收软引用
        TimeUnit.SECONDS.sleep(1L);
        System.out.println("gc-Before 内存够用 :" + softReference.get());
        try {
            byte[] bytes = new byte[20 * 1024 * 1024];	//占满内存
        } catch (Exception e) {
            e.printStackTrace();
        } finally {
            // null 内存不足被自动回收
            System.out.println("gc-After 内存不够:" + softReference.get());
        }
    }
}
```

>
> gc-After 内存不够:null
>
> 即将调用gc回收无用对象~

## 弱引用
:::

弱引用比软引用生存周期更短，只要运行垃圾回收机制，不管内存是否充足，<font style="color:#1DC0C9;">都会回收该对象</font>。

```java
class Paper {
    //gc回收之前会自动调用该方法 一般不需要自己重写
    @Override
    protected void finalize() throws Throwable {
        System.out.println("即将调用gc回收无用对象~");
    }
}

public class gcDemo {
    public static void main(String[] args) throws InterruptedException {

        WeakReference<Paper> weakReference = new WeakReference<>(new Paper());
        System.out.println("gc之前:" + weakReference.get());
        System.gc();
        TimeUnit.SECONDS.sleep(1L);
        // null 由于是弱引用会被回收
        System.out.println("gc之后" + weakReference.get());
    }
}
```

## 虚引用
:::

虚引用就是<font style="color:#ECAA04;">形同虚设</font>，与其他引用不同，虚引用不决定对象的生命周期。如果<font style="color:#117CEE;">一个对象仅持有虚引用，那么它就和没有任何引用一样，在任何时候对会被垃圾回收器回收，它不能单独使用也不能通过他访问对象</font>，<font style="color:#74B602;">虚引用必须和引用队列（ReferenceQueue）联合使用。</font>

<font style="color:#74B602;"></font>

<font style="color:#ECAA04;">作用</font>：仅提供一种确保对象被finalize以后，做某些逻辑的通知机制PhantomReference的get方法总是返回null，因此无法访问对应的引用对象。

<font style="color:rgb(102, 102, 102);background-color:#EFF0F0;">当垃圾回收器准备回收一个对象时，如果发现它还有虚引用，就会在回收对象的内存之前，把这个虚引用加入到与之关联的引用队列中。</font>

```java
class Paper {}

public class gcDemo {
    public static void main(String[] args) throws InterruptedException {

        ReferenceQueue<Paper> referenceQueue = new ReferenceQueue<>();	//引用队列
        
        //创建一个虚引用，并关联一个引用队列
        PhantomReference<Paper> phantomReference = new PhantomReference<>(new Paper(), referenceQueue);	

        List<byte[]> list = new ArrayList<>();

        new Thread(() -> {
            while(true) {
                list.add(new byte[1024 * 1024]);	//占满内存使gc自动回收之前创建的虚引用
                try {
                    TimeUnit.MILLISECONDS.sleep(500L);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
                System.out.println(phantomReference.get() + "\t" + "list add ok");	//一直返回null
            }
        },"t1").start();

        new Thread(() -> {
            while(true) {
                //gc回收一个对象时发现该对象存在虚引用且关联了引用队列则会在回收前将虚引用加入该队列中
                Reference<? extends Paper> poll = referenceQueue.poll();	//从队列中取出被加入的虚引用或空值
                if (poll != null) {	//验证是否有虚引用被加入队列
                    System.out.println("有虚引用加入引用队列");
                    break;
                }
            }
        },"t2").start();
    }
}
```
