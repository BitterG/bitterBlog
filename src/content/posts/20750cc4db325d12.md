# frida gadget加载so实现无root的APP HOOK

## 1.下载所需文件

github frida仓库 <https://github.com/frida/frida>下载frida-gadget-16.7.19-android-arm64.so.xz或其他对应所需架构so

## 2.解包(以无加固为例)

这里使用了安卓官方提供的<https://apktool.org/blog>工具

解包 d表示解包

> java -jar apktool\_2.12.0.jar d xxx.apk

使用直接zip的解压可能导致文件命名出现问题，打包重签名后无法正常运行。

## 3.加工SO

取出lib中的app原本的so(注意要保证so是可被加载的，否则我们设置的依赖so是无法跟随依赖库被加载)。

我这里使用linux(ubuntu22.04)环境下使用patchelf将frida-gadget.so设置为目标so的依赖库。

> patchelf --add-needed frida-gadget.so apk-target.so

frida-gadget.so是我们从frida仓库下载的，apk-target.so是app中原有的

设置依赖库后使用IDA查看前后区别，使用patchelf添加后多出一个frida-gadget.so的依赖项(我们这里对frida-gadget.so进行了重命名后再设置依赖库防止检测，命名规范要求以lib开头，.so结尾)

## 4.编写so对应配置文件

这里采用gadget的script模式加载脚本<https://frida.re/docs/gadget/>

查看文档最简单配置如下，"path"是脚本存放路径，可将脚本放在其他可读取位置，例如/data/local/tmp/xxx.js

![图片](/img/frida/20750cc4db325d12/1757753015335-85a5ddfd-f5c1-4d0a-a588-bbc2d7e5b647-975882.png)

```javascript
{
  "interaction": {
    "type": "script",
      "path": "/data/local/tmp/hook.js"
  }
}
```

文档要求配置文件与.so文件放在一起，且命名为xxx.config.so,xxx为我们重命名的frida-gadget.so

![图片](/img/frida/20750cc4db325d12/1757753249248-85a92c00-e607-4546-8ef0-cff876b325ff-345060.png)

## 5.重打包

使用apktool进行重打包

b表示打包选项 app-release为原app-release.apk经apktool解包自动生成的目录，-o表示输出新apk的位置

> java -jar apktool\_2.12.0.jar b app-release -o new.apk

打包目标是API 30(android 13)以上的版本可能会出现资源缺失(color,value-v31等)，和不识别的属性在AndroidManifest.xml中。

解决方案：

1.AndroidManifest.xml不识别属性是apktool解包时生成的根据终端报错在AndroidManifest.xml中删去对应属性即可。

2.资源缺失问题，前往官方下载对应版本镜像<https://developer.android.google.cn/topic/generic-system-image/releases>找到其中的system.img，我这里在ubuntu下对镜像进行挂载(部分镜像可能需要格式转换)

> sudo mount -t ext4 -o loop,ro system.img sys/

![图片](/img/frida/20750cc4db325d12/1757754286306-3cc83874-820b-4f57-a89e-3f7a5eb67b21-555635.png)

取出其中的/sys/system/framework/framework-res.apk，使用apktool进行安装缺失的属性

> java -jar apktool.jar if framework-res.apk

安装后重打包，可能还会缺失部分属性我们根据报错找到对应部分进行属性替换(不建议直接删除)，如使用同类可识别属性替换缺失的，以\res\values-v31\colors.xml缺失资源为例![图片](/img/frida/20750cc4db325d12/1757754692491-ca3dc6bd-bd81-4ca6-a64b-b848b07911f6-373994.png)

## 6.签名

没证书可使用JDK自带keytool进行生成

> keytool -genkeypair -v -keystore my-release-key.jks -keyalg RSA -keysize 2048 -validity 10000 -alias mykey

* `-genkeypair`：生成一个密钥对（公钥+私钥）。
* `-keystore my-release-key.jks`：生成的证书文件名（就是 `.jks` 文件）。
* `-keyalg RSA`：加密算法，安卓一般用 `RSA`。
* `-keysize 2048`：密钥长度，推荐 2048。
* `-validity 10000`：有效期，单位是天，这里是 27 年左右。
* `-alias mykey`：别名，后续签名 APK 时要用到。

使用Android\Sdk\build-tools\版本\ 下的apksigner进行签名

> PS C:\Users\kugua\AppData\Local\Android\Sdk\build-tools\35.0.0> .\apksigner.bat sign --ks 证书路径.jks --out 签完生成的路径.apk 之前重打包的路径.apk
