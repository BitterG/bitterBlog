# 1.NIO的起步

Nio引入两个新的概念<font style="color:#1DC0C9;">通道</font>和<font style="color:#1DC0C9;">缓冲区</font>，Nio的操作都要依靠这两个东西。



通道是堆传统I/O包中流的模拟，数据的读写都要经过通道（Channel对象）。

缓冲区（Buffer）就是一个容器，数据读写都要经过缓冲区，读：数据->通道->缓冲区，写：缓冲区->通道->输出数据。



<font style="color:rgb(18, 18, 18);">缓冲区实质上是一个数组。通常它是一个字节数组，但是也可以使用其他种类的数组。但是一个缓冲区不 仅仅 是一个数组。缓冲区提供了对数据的结构化访问，而且还可以跟踪系统的读/写进程。</font>

<font style="color:rgb(18, 18, 18);"></font>

<font style="color:rgb(18, 18, 18);">每一个 Buffer 类都是 Buffer 接口的一个实例。 除了 ByteBuffer，每一个 Buffer 类都有完全一样的操作，只是它们所处理的数据类型不一样。因为大多数标准 I/O 操作都使用 ByteBuffer，所以它具有所有共享的缓冲区操作以及一些特有的操作。</font>

## <font style="color:rgb(18, 18, 18);">使用步骤</font>
1.从流获取Channel

2.创建Buffer

3.将数据从Channel读到Buffer

## 读取文件
例子：

<font style="color:rgb(18, 18, 18);">第一步是获取通道。我们从 FileInputStream 获取通道：</font>

> <font style="color:rgb(99, 99, 99);">FileInputStream fin = new FileInputStream( "readandshow.txt");  
</font><font style="color:rgb(99, 99, 99);">FileChannel fc = fin.getChannel();</font>
>

<font style="color:rgb(18, 18, 18);">下一步是创建缓冲区：</font>

> <font style="color:rgb(99, 99, 99);">ByteBuffer buffer = ByteBuffer.allocate(1024);</font>
>

<font style="color:rgb(18, 18, 18);">最后，需要将数据从通道读到缓冲区中，如下所示：</font>

> <font style="color:rgb(99, 99, 99);">fc.read( buffer );</font>
>

<font style="color:rgb(18, 18, 18);">我们不需要告诉通道要读 多少数据 到缓冲区中。每一个缓冲区都有复杂的内部统计机制，它会跟踪已经读了多少数据以及还有多少空间可以容纳更多的数据。</font>

## <font style="color:rgb(18, 18, 18);">写入文件</font>
<font style="color:rgb(18, 18, 18);">在 NIO 中写入文件类似于从文件中读取。首先从 FileOutputStream 获取一个通道：</font>

> <font style="color:rgb(99, 99, 99);">FileOutputStream fout = newFileOutputStream( "writesomebytes.txt");  
</font><font style="color:rgb(99, 99, 99);">FileChannel fc = fout.getChannel();</font>
>

<font style="color:rgb(18, 18, 18);">下一步是创建一个缓冲区并在其中放入一些数据 - 在这里，数据将从一个名为 message 的数组中取出，这个数组包含字符串 "Some bytes" 的 ASCII 字节(后面将会解释 buffer.flip() 和 buffer.put() 调用)。</font>

> <font style="color:rgb(99, 99, 99);">ByteBuffer buffer = ByteBuffer.allocate( 1024);  
</font>
>
> <font style="color:rgb(99, 99, 99);">//这里使用put方法将数据放入缓冲区  
</font><font style="color:rgb(99, 99, 99);">for(intii=0; ii<message.length; ++ii) {  
</font><font style="color:rgb(99, 99, 99);">	buffer.put( message[ii] );  
</font><font style="color:rgb(99, 99, 99);">}  
</font><font style="color:rgb(99, 99, 99);">buffer.flip();</font>
>

<font style="color:rgb(18, 18, 18);">最后一步是写入缓冲区中：</font>

> <font style="color:rgb(99, 99, 99);">fc.write( buffer );</font>
>

<font style="color:rgb(18, 18, 18);">注意在这里同样不需要告诉通道要写入多数据。缓冲区的内部统计机制会跟踪它包含多少数据以及还有多少数据要写入。</font>

## <font style="color:rgb(18, 18, 18);">读取状态检查</font>
> <font style="color:#000000;">channel.read(byteBuffer);</font>
>

<font style="color:rgb(18, 18, 18);">当没有更多的数据时，拷贝就算完成，并且可以在 read() 方法返回 -1 是判断这一点</font>

<font style="color:rgb(18, 18, 18);">例子：</font>

```java
//从文件读取到缓冲区
int i = channel.read(byteBuffer);
//判断是否读到数据
if (i == -1) break;
```

<font style="color:rgb(18, 18, 18);">clear() 方法重设缓冲区，使它可以接受读入的数据。 flip() 方法让缓冲区可以将新读入的数据写入另一个通道。（详细请看2.缓冲区细节部分）</font>

<font style="color:rgb(18, 18, 18);"></font>

```java
int i = 0;
while (i != -1) {
    //从文件读取到缓冲区
    i = channel.read(byteBuffer);
    //判断是否读到数据
    if (i == -1) break;

    //从缓冲区拿出数据写文件
    byteBuffer.flip();
    channelOut.write(byteBuffer);

    //写完清空缓冲区
    byteBuffer.clear();
}
```

<font style="color:#1DC0C9;">read() 和 write() 调用得到了极大的简化，因为许多工作细节都由缓冲区完成了。 clear() 和 flip() 方法用于让缓冲区在读和写之间切换。</font>

<font style="color:#1DC0C9;"></font>

<font style="color:#1DC0C9;"></font>

<font style="color:#000000;">一个完整的读取并写出文件的例子</font>

```java
package com.bitter.nio;

import org.junit.platform.commons.util.StringUtils;
import org.mockito.internal.util.StringUtil;

import java.io.*;
import java.nio.ByteBuffer;
import java.nio.channels.FileChannel;

public class NioDemo {

    public static void main(String[] args) {

        FileInputStream inputStream = null;
        FileChannel channel = null;
        FileOutputStream outputStream = null;
        FileChannel channelOut = null;

        try {
            inputStream = new FileInputStream("C:\\Users\\user\\Desktop\\GQ.java");
            channel = inputStream.getChannel();

            outputStream = new FileOutputStream("C:\\Users\\user\\Desktop\\GQ_Copy.java", true);
            channelOut = outputStream.getChannel();

            ByteBuffer byteBuffer = ByteBuffer.allocate(1024);

            int i = 0;
            while (i != -1) {
                //从文件读取到缓冲区
                i = channel.read(byteBuffer);
                //判断是否读到数据
                if (i == -1) break;

                //从缓冲区拿出数据写文件
                byteBuffer.flip();
                channelOut.write(byteBuffer);

                //写完清空缓冲区
                byteBuffer.clear();
            }

            channel.close();
            channelOut.close();
            outputStream.close();
            inputStream.close();

        } catch (IOException e) {
            e.printStackTrace();
        }

    }

}
```
